# Xiaomi 12 — GPS in EXIF

Pipeline reads GPS from photo EXIF (lat/lon). Without GPS the tile renders without a 📍 / `↗` link. This doc covers (1) saving GPS at capture and (2) injecting GPS retroactively into photos that lack it.

Tested on Xiaomi 12 / MIUI 14 / HyperOS 1. Menu wording shifts slightly between versions.

## 1. Save GPS at capture (the easy way)

Four toggles must all be ON. If any is OFF, no GPS in EXIF.

### a. OS location

Settings → **Location** → toggle ON. Mode: "Use GPS, Wi-Fi & networks" (high accuracy).

### b. Camera app permission

Settings → **Apps** → **Manage apps** → **Camera** → **Permissions** → **Location** → **Allow all the time** (or "Allow only while using app").

### c. Camera in-app toggle

Open Camera → ⚙ (top gear) → scroll → **"Save location info"** / **"Standortinformationen speichern"** → ON.

### d. Privacy strip

Settings → **Privacy protection** (Datenschutz) → **"Empty location info"** must be **OFF**. (When ON it strips GPS from any image you share. It doesn't affect the file in DCIM, but if you transfer via the share-sheet it may pass through this filter.)

### GPS lock

Outdoors. First fix takes 10-30 s after opening Camera. Indoors GPS often won't lock — phone falls back to Wi-Fi/cell triangulation if available, otherwise no GPS.

Tip: open **Maps** briefly before each shot to refresh fix. Camera reuses the last lock.

### Verify on phone

Gallery → tap photo → swipe up / ⓘ / **Details** → location pin / coordinates shown = OK.

### Verify on PC

```powershell
python -c "from PIL import Image,ExifTags; t=next(k for k,v in ExifTags.TAGS.items() if v=='GPSInfo'); g=Image.open('raw/042.jpg')._getexif().get(t); print('GPS OK' if g else 'NO GPS')"
```

### Transfer without losing GPS

Compression strips EXIF. Safe transfers:

| Method | EXIF preserved? |
|---|---|
| USB cable (MTP) | ✅ |
| Mi Mover / MiShare | ✅ |
| WhatsApp "Document" mode | ✅ |
| Telegram "Send as file" | ✅ |
| Email attachment (original size) | ✅ |
| WhatsApp default photo send | ❌ stripped |
| Telegram default photo send | ❌ stripped |
| Instagram / WhatsApp Status | ❌ stripped |

When in doubt, USB cable.

## 2. Inject GPS retroactively (when capture missed it)

Sometimes a photo lacks GPS — captured indoors, or transferred through a stripping app. Add GPS by hand using **ExifTool** (works on Windows, free).

### Install

```powershell
winget install --id OliverBetz.ExifTool
```

(or download from https://exiftool.org/ and unzip; rename `exiftool(-k).exe` → `exiftool.exe`).

### Find the coordinates

Open Google Maps, right-click the door's exact location → click the lat/lon line at top → coordinates copied to clipboard. Format: `49.234567, 7.012345`.

### Inject

```powershell
exiftool -GPSLatitude=49.234567 -GPSLatitudeRef=N -GPSLongitude=7.012345 -GPSLongitudeRef=E raw\042.jpg
```

Saarbrücken is always **N** (north of equator) and **E** (east of Greenwich), so refs are constant for this project. Negative latitudes need `-GPSLatitudeRef=S`, negative longitudes `-GPSLongitudeRef=W`.

ExifTool writes a backup `raw\042.jpg_original` by default. After verifying, delete the backup or pass `-overwrite_original` next time:

```powershell
exiftool -overwrite_original -GPSLatitude=49.234567 -GPSLatitudeRef=N -GPSLongitude=7.012345 -GPSLongitudeRef=E raw\042.jpg
```

### Verify

```powershell
exiftool -GPSPosition raw\042.jpg
```

Expected:

```
GPS Position : 49 deg 14' 4.44" N, 7 deg 0' 44.44" E
```

Then re-process and re-bake:

```powershell
python process.py --only 42 --force
python bake.py --inject gallery.html
```

### Bulk inject from a CSV

If you collected coordinates separately (e.g. notebook entries during the walk), save as `gps.csv`:

```
slot,lat,lon
007,49.2401,7.0035
015,49.2335,7.0102
022,49.2298,6.9981
```

then loop:

```powershell
Import-Csv gps.csv | ForEach-Object {
  $f = "raw\$('{0:000}' -f [int]$_.slot).jpg"
  exiftool -overwrite_original "-GPSLatitude=$($_.lat)" -GPSLatitudeRef=N "-GPSLongitude=$($_.lon)" -GPSLongitudeRef=E $f
}
python process.py --force
python bake.py --inject gallery.html
```

## Common failure modes

| Symptom | Cause |
|---|---|
| `bake.py` warns `no GPS - shown without map pin` | Capture missed it (indoor, no fix) or transfer stripped it. Inject via ExifTool. |
| GPS pin off by 50-200 m | Indoor / urban canyon / Wi-Fi triangulation only. Acceptable for door discovery. To fix exactly: re-shoot outdoors or override via ExifTool with Maps coordinates. |
| GPS pin in wrong country | Camera locale/sign error. Check `-GPSLatitudeRef` (N for Saarbrücken) and that latitude is positive. |
| Photo shows GPS in MIUI Gallery but `bake.py` doesn't see it | Pillow EXIF reader can fail on certain MIUI quirks. Re-save through ExifTool: `exiftool -all= -tagsfromfile @ -all:all -unsafe raw\042.jpg` (this rewrites a clean EXIF block). |
